# Arena Platform - Tournament Dashboard

A premium, high-performance esports tournament and asset management dashboard designed for competitive gaming ecosystems. The platform features an integrated responsive navigation structure, real-time reactive wallet metrics, persistent global data states, smooth context-driven dark/light mode toggles, and customized cyberpunk design elements.

🎯 **[Live Demo]()**

---

## 🚀 Key Features

* **Dynamic Navigation Shell:** Responsive desktop static sidebar that seamlessly transforms into a mobile drawer overlay complete with automatic closing triggers on item selection.
* **Wallet & Asset Ledger:** Programmatic balance management supporting instant multi-mode asset transactions (Deposits & Withdrawals) with input safety guards.
* **Symmetrical Grid Interfaces:** Real-time layout height equalization utilizing CSS flex-stretch boundaries alongside optimized text truncation mechanisms.
* **Persistent State Synchronization:** Global state preservation tracking multi-game active tournament registrations and structural transaction logs via unified store slices.
* **Cyberpunk Aesthetics & Theme Control:** Deep dark default UI modes built using modern OKLCH color palettes, featuring custom-designed minimalist scrollbars and default browser control overrides.

## 🛠️ Tech Stack

### Core Framework & Routing
* **React 19** - Functional UI rendering engine utilizing concurrent feature pipelines.
* **TypeScript 6** - Strict compile-time static layout structural typing.
* **React Router DOM v7** - Client-side layout routing tree configurations and parameter matching.

### State Management & Utilities
* **Zustand v5** - Lightweight, atomic global state machine utilizing continuous local cache persistence layers.
* **React Context API** - PRESENTATIONAL UI thematic state provider for localized app cascading.

### Styling & Components
* **Tailwind CSS v4** - Performance-focused utility compiler operating with modern `@theme inline` oklch setups.
* **Vite Tooling** - Optimized compiler integration bundle mappings.
* **shadcn/ui & Radix Primitives** - Accessible, unstyled functional runtime UI component shells.
* **Lucide React** - Minimalist vector UI layout iconography.
* **React Hot Toast** - Programmatic notification layer hook system.

## 💻 Getting Started & Set Up

Follow these steps to configure your local development workspace environment:

### Prerequisites
Ensure you have the following runtime tool configuration setups installed globally on your machine:
* **Node.js** (v18.x or higher recommended)
* **npm** (v9.x or higher)

### 1. Clone the Repository
```bash
git clone https://github.com/danluchin1/tournament-dashboard
cd tournament-dashboard
```

### 2. Install Project Dependencies
Run the installation command to fetch all designated frame dependencies listed inside the configuration manifest file:

```bash
npm install
```

### 3. Run the Local Development Server
Launch the local development workspace engine. Vite compiles the operational links onto a default development port target:

```bash
npm run dev
```

Once initialized, open your browser and navigate to the application terminal window:
👉 http://localhost:5173

### 4. Available Production & Validation Scripts
You can trigger any of these designated package command vectors inside your terminal stream:

*   **npm run dev** - Launches the reactive Vite live reload engineering workspace environment.
*   **npm run build** - Executes strict TypeScript compilation (tsc) and builds optimized, production-ready bundle distribution assets.
*   **npm run lint** - Code analyzer checks tracing style optimizations via standard ESLint rulesets.
*   **npm run preview** - Runs a static preview server tracing built production outputs locally before cloud deployment.
